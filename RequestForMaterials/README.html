1. このシステムは何か（概要）

議員が議長（議会事務局）あてに行う「資料要求」の申請を、紙の窓口提出に代えてオンラインで受け付けるためのWebフォームです。 議員がスマートフォンやパソコンから申請すると、内容がGoogleスプレッドシートに記録され、議員本人と議会事務局の双方に自動でメールが届きます。

業務フロー上の位置づけ

議員 ──[本システムで申請]──> 議長（議会事務局） ──[従来どおり取り次ぎ]──> 首長（執行部局） ↑ここをオンライン化 ↑ここから先は本システムの範囲外

主な機能

  • 申請フォーム：氏名・メールアドレス・資料の内容・担当部署・調整状況（未調整／調整済）・用途・希望提出期限を入力して送信
  • 本人確認：事前に「議員名簿」シートへ登録したメールアドレス以外からは申請できない
  • 受付番号の自動発行：20260610-001 形式（日付＋その日の連番）
  • 自動メール通知：議員へ受付確認メール、議会事務局へ新規申請の通知メール
  • 過去申請の照会：議員がメールアドレスを入力すると自分の申請履歴を一覧できる別ページ

意図的に「持っていない」機能

  • パスワードによるログイン認証（メールアドレス照合のみ）
  • 申請のステータス管理（処理中・完了など。議員側に進捗を見せない運用方針のため）
  • 申請の取消・修正機能（必要な場合は事務局がスプレッドシートを直接編集する）

2. 仕組み（アーキテクチャ）

本システムはGoogle Apps Script（GAS）のWebアプリ機能だけで動いています。 サーバーの契約・保守は不要で、費用はかかりません。Googleアカウント（スプレッドシート＋GAS）がすべてです。

┌─ 議員のスマホ／PC（ブラウザ）─────────────────┐ │ フォームページ（index）／照会ページ（history） │ └──────────────┬───────────────┘ │ google.script.run（GAS内部通信） ┌──────────────▼───────────────┐ │ Google Apps Script（Code.gs） │ │ ・登録アドレスの照合 ・受付番号の採番 │ │ ・データの保存 ・メール送信 │ └──────────────┬───────────────┘ │ ┌──────────────▼───────────────┐ │ Googleスプレッドシート（3シート） │ │ 申請データ／議員名簿／設定 │ └──────────────────────────────┘

重要な考え方

  • スプレッドシートが「データベース」かつ「管理画面」です。事務局の日常操作は基本的にスプレッドシートの編集だけで完結します。
  • プログラム（Code.gs）は、スプレッドシートに紐付いた「コンテナバインド型」のGASです。スプレッドシートを開き、メニューの「拡張機能 → Apps Script」から開けます。
  • Webアプリは「実行ユーザー＝デプロイした人のアカウント」で動きます。メールもそのアカウントから送信されます。したがって個人アカウントではなく、組織の共用アカウント（例：議会事務局の代表Googleアカウント）で構築・公開することを強く推奨します。
⚠ 属人化を防ぐ最重要ポイント スプレッドシートとGASの「所有者」が退職・異動した個人アカウントのままだと、アカウント削除と同時にシステムが消滅します。 必ず共用アカウントの所有とするか、後任への所有権移転を引き継ぎ事項に含めてください（→ 9. 引き継ぎチェックリスト）。

4. 初期構築手順（ゼロから作る場合）

システムを新規に構築する場合、または作り直す場合の手順です。所要時間は30分程度です。

4-1. スプレッドシートとGASプロジェクトの作成

  1. 組織の共用Googleアカウントでログインする
  2. Googleスプレッドシートを新規作成し、名前を「議員資料要求フォーム（データ）」等にする
  3. メニュー「拡張機能」→「Apps Script」を開く
  4. 最初からある コード.gs の中身をすべて消し、Code.gs の内容を貼り付ける
  5. 左側「ファイル」の「＋」→「HTML」で index という名前のファイルを作成し、index.html の内容を貼り付ける（拡張子 .html は自動で付く）
  6. 同様に history を作成し、history.html の内容を貼り付ける
  7. 保存（Ctrl+S または 💾アイコン）

4-2. シートの初期化

  1. GASエディタ上部の関数選択プルダウンで setup を選び、「実行」をクリック
  2. 初回は「承認が必要です」と表示される。「権限を確認」→ アカウント選択 →（「確認されていないアプリ」と出た場合は「詳細」→「（安全ではないページ）に移動」）→「許可」
  3. スプレッドシートに「申請データ」「議員名簿」「設定」の3シートが生成されたことを確認
  4. 議員名簿シートの見本行（example.lg.jp）を削除し、実際の議員のメールアドレスと氏名を登録
  5. 設定シートのB2セルを、実際の事務局通知先メールアドレスに変更

4-3. Webアプリとして公開

  1. GASエディタ右上「デプロイ」→「新しいデプロイ」
  2. 歯車アイコン →「ウェブアプリ」を選択
  3. 設定値：
    項目設定値
    説明任意（例：初回公開）
    次のユーザーとして実行自分
    アクセスできるユーザー全員
  4. 「デプロイ」をクリックし、発行された ウェブアプリのURL を控える
  5. 本書3章のURL欄に記入し、議員へ案内する

4-4. 動作確認（公開前に必ず実施）

  1. 議員名簿にテスト用に自分のアドレスを一時登録する
  2. フォームURLを開き、全項目を入力して送信 → 受付番号が表示されるか
  3. 「申請データ」シートに1行追加されているか
  4. 自分宛の受付確認メールと、設定シートの通知先宛のメールが届いているか（迷惑メールフォルダも確認）
  5. 未登録のアドレスで送信し、「登録されていないメールアドレスです。」と拒否されるか
  6. 照会ページ（?page=history）で自分のアドレスを入力し、テスト申請が表示されるか
  7. 確認後、テスト用アドレスを名簿から削除し、テストデータの行を申請データから削除する

5. 日常の運用作業

通常運用で事務局が行う作業は、以下の3つだけです。いずれもスプレッドシートの編集のみで、プログラムの変更は不要です。

5-1. 申請の確認と取り次ぎ

  • 新規申請があると、設定シートに登録した通知先へメールが届きます（件名：【資料要求】新規申請）。
  • メール本文に全項目が記載されているため、原則メールだけで内容確認できます。一覧で見たい場合は「申請データ」シートを開いてください。
  • その後の議長決裁・執行部局への要求は、従来どおりの事務処理で行ってください（本システムの範囲外）。
  • 「調整状況」列は、議員が担当部署と事前調整済みかどうかの自己申告です。「未調整」の案件は取り次ぎ前に調整の要否を判断してください。

5-2. 議員名簿の更新（改選時・随時）

  1. 「議員名簿」シートを開く
  2. 追加：最終行の下に「メールアドレス」「氏名」を入力（順不同・空行は不可）
  3. 削除：退任議員の行を行ごと削除
  4. 保存操作は不要（スプレッドシートは自動保存）。即時反映されます。
✓ 補足 メールアドレスの大文字・小文字、前後の空白は自動で吸収されるため、厳密に揃える必要はありません。1人の議員が複数アドレスを使う場合は、複数行に分けて登録できます（氏名は同じでかまいません）。

5-3. 通知先メールアドレスの変更（担当変更時）

  1. 「設定」シートを開く
  2. 「議会事務局通知先メールアドレス」の行のB列を書き換える
  3. 複数に送りたい場合はカンマ区切りで記入（例：a@example.lg.jp, b@example.lg.jp）

5-4. データの保全（推奨・月1回程度）

  • スプレッドシートの「ファイル」→「ダウンロード」→「Microsoft Excel」で控えを保存し、所定の共有フォルダに保管してください。
  • 年度替わり等で申請データが増えすぎた場合は、過去年度分を別ファイルに切り出してから行を削除して構いません（受付番号は日付ベースのため連番が崩れる心配はありません）。ただし削除した分は照会ページに表示されなくなる点に留意してください。

6. よくあるトラブルと対処

症状原因と対処
議員から「登録されていないメールアドレスですと出る」と連絡 議員名簿に当該アドレスが未登録、または誤記。名簿のアドレスと議員が入力したアドレスを突き合わせる。よくあるのは携帯キャリアアドレスとGmailの取り違え、ドメインの打ち間違い。
確認メールが議員に届かない ①迷惑メールフォルダを確認させる。②携帯キャリアメールの場合、PCメール拒否設定で弾かれることがある → 送信元（デプロイしたGoogleアカウントのGmailアドレス）の受信許可を依頼。③申請自体は「申請データ」シートに記録されていれば受付済みなので、その旨を案内してよい。
事務局に通知メールが届かない 「設定」シートのB列のアドレスに誤記がないか確認。シート名が「設定」、A列の項目名が「議会事務局通知先メールアドレス」と一字一句一致しているかも確認（プログラムはこの文字列で行を探している）。
フォームのURLを開くと「権限が必要です」「このアプリは確認されていません」等が表示される デプロイ設定の「アクセスできるユーザー」が「全員」になっていない可能性。デプロイ → デプロイを管理 → 編集（鉛筆）で設定を確認し、バージョンを「新バージョン」にしてデプロイし直す。
フォームを開くと真っ白／エラー画面になる GASの障害または直近のコード修正ミスの可能性。GASエディタの「実行数」（時計アイコン）でエラーログを確認。直近に修正をしていた場合は、デプロイを管理から前のバージョンに戻すことができる。
1日の途中からメールが送られなくなった Googleアカウントの1日あたりメール送信上限（無料アカウント100通／Google Workspace 1,500通）に達した可能性。翌日自動的に回復する。申請データ自体は記録されている。
同じ受付番号が重複した 通常は排他制御により発生しないが、万一発生した場合はスプレッドシート上で手動で枝番（例：-001a）を振って区別し、関係者に連絡する。

エラーログの見方

  1. スプレッドシート →「拡張機能」→「Apps Script」
  2. 左メニューの時計アイコン「実行数」をクリック
  3. 「失敗」となっている行を開くと、エラー内容と発生箇所が表示される

7. プログラムを修正・再公開する手順

7-1. 修正の基本手順

  1. GASエディタで該当ファイル（Code.gs / index / history）を修正し、保存
  2. 右上「デプロイ」→「デプロイを管理」
  3. 既存のデプロイの鉛筆アイコン（編集）をクリック
  4. 「バージョン」を「新バージョン」に変更し、「デプロイ」
✗ やってはいけない 「新しいデプロイ」から公開するとURLが変わってしまい、議員に案内済みのURLが使えなくなります。修正の反映は必ず「デプロイを管理」→ 既存デプロイの編集で行ってください。
✓ 失敗してもすぐ戻せます 「デプロイを管理」の編集画面でバージョン番号を過去のものに選び直してデプロイすれば、即座に以前の状態へ戻せます。修正前のバージョン番号を控えておくと安心です。

7-2. よくある軽微な修正の場所

変更したいこと修正場所
フォームの文言・項目名・注意書きindex 内のHTML部分（<label>や<p class="hint">の文章）
メール本文の文面Code.gs の buildMailBody_ 関数
メールの件名Code.gs の sendApplicantMail_（議員宛）／sendOfficeMail_（事務局宛）
受付番号の形式Code.gs の generateReceiptNumber_ 関数
入力項目の追加・削除index（画面）と Code.gs（COL定数・submitRequest・ヘッダー行）の両方。シートの列追加も必要。難易度が高いため7-3の方法を推奨。

7-3. 生成AIを活用した改修（推奨）

本システムは、3ファイルすべてを生成AI（Claude等）に貼り付ければ全体を把握できる規模で設計されています。 改修したい場合は、次の手順が安全かつ確実です。

  1. Code.gs / index / history の3ファイルの中身と本マニュアルの該当部分をAIに渡す
  2. 「○○という項目を追加したい」「文言を△△に変えたい」等の要望を伝える
  3. 修正版のコードを受け取り、テスト用にシートを複製した別環境で動作確認してから本番へ反映する

8. セキュリティと運用上の割り切り

本システムは利便性を優先し、以下を承知のうえで運用しています。後任者は前提として把握してください。

  • 認証はメールアドレス照合のみです。パスワードはありません。なりすまし防止は「登録アドレス以外は受け付けない」ことのみに依存しています。
  • 照会ページは、議員のメールアドレスを知っていれば誰でもその議員の申請履歴を閲覧できます。機微な内容を含む資料要求が想定される場合は、運用ルール（紙での提出に切り替える等）でカバーするか、受付番号との二要素照合方式への改修を検討してください。
  • フォームURL自体は非公開ですが、URLを知っていれば誰でもフォーム画面の表示までは可能です（送信は登録アドレスのみ）。URLは議員と事務局以外に共有しないでください。
  • 申請データには議員の氏名・メールアドレスが含まれます。スプレッドシートの共有設定は事務局の必要最小限の職員のみとし、「リンクを知っている全員」には絶対にしないでください。
  • メール送信はGoogleアカウントの送信上限（無料100通/日、Workspace 1,500通/日）の範囲内で動作します。

ソースファイルの保管について

GASエディタ上のコードが「本番」ですが、不測の事態（誤削除等）に備え、 Code.gs / index.html / history.html の3ファイルと本マニュアルを、必ず庁内の共有フォルダ等にも保管してください。 この4ファイルさえあれば、本書4章の手順でシステム全体をいつでも再構築できます。
