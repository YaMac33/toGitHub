会議室予約申請システム

HTMLフォーム（フロントエンド）と Google スプレッドシート＋Google Apps Script（バックエンド）で構成される、 サーバーレスの会議室予約申請システムです。外部サービスへの依存なしに庁内・組織内で即運用できます。

HTML / CSS / JavaScript Google Apps Script サーバーレス 外部依存なし 要 Google アカウント

1ファイル構成

ファイル名種別役割
meeting-room-form.htmlフロントエンド申請フォーム本体。ブラウザ上で動作する単一HTMLファイル。スタイル・ロジックをすべて内包。
meeting-room-gas.jsバックエンド（GAS）Google Apps Script のコード。申請受信・スプレッドシート書き込み・メール通知・公開データ同期を担う。
README.htmlドキュメント本ファイル。システム仕様・セットアップ手順・運用方法を記述。

2アーキテクチャ

本システムはすべて Google のインフラ上で完結します。専用サーバーや有料サービスは不要です。

レイヤー技術説明
フロントエンドHTML / CSS / JavaScript（単一ファイル）申請フォーム・確認画面・予約状況表示。外部ライブラリ不使用（Google Fonts のみ）。
API層Google Apps Script（ウェブアプリ）POST（申請受信）/ GET（予約一覧取得）エンドポイントを提供。
データ層Google スプレッドシート申請データ・公開データ・設定を3シートで管理。
通知GmailApp（GAS内蔵）申請受信時に管理者へメール送信。
ℹ️
GAS の制限について：無料の Google アカウントでは、GAS の実行時間・メール送信数などに上限があります（1日のメール送信上限：100通）。大量申請が見込まれる場合は Google Workspace アカウントの利用を検討してください。

3セットアップ手順

① スプレッドシートの準備

Google スプレッドシートを新規作成し、以下の3つのシートを作成します（シート名は完全一致が必要です）。

シート名用途備考
申請データ全申請情報を記録。管理者が操作するシート。ステータス列にプルダウンが自動設定される
公開データフロントの予約状況エリアが参照するシート。GASが自動同期。手動編集不要。
設定通知先メールアドレス等の設定値を管理。B1セルにメールアドレスを記入
⚠️
シート名は全角・半角を含め完全一致：「申請データ」「公開データ」「設定」のシート名が GAS コード内の定数と一致しないと動作しません。コピペで作成することを推奨します。

② initSheets 関数の実行（初回のみ）

スクリプトエディタを開き、initSheets 関数を手動実行します。これにより各シートのヘッダー行・プルダウン設定が自動作成されます。

  1. 1
    スプレッドシートのメニューから 「拡張機能」→「Apps Script」 を開く
  2. 2
    GASコード（meeting-room-gas.jsの内容）をエディタに貼り付け、保存
  3. 3
    関数名の選択プルダウンで 「initSheets」 を選択し、▶ 実行ボタンをクリック
  4. 4
    初回は権限の確認ダイアログが表示されます。「権限を確認」→ Googleアカウントを選択 →「詳細」→「安全ではないページに移動」→「許可」 の順に進めてください
  5. 5
    「初期化が完了しました。」のアラートが表示されれば成功
✅
initSheets 実行後の状態：「申請データ」シートに12列のヘッダー行が作成され、K列（ステータス）に「確認中 / 予約確定 / 申請却下」のプルダウンが設定されます。「公開データ」「設定」シートにもヘッダーと初期値が作成されます。

③ 設定シートへの記入

セル項目名記入内容例
A1通知先メールアドレス（ラベル）変更不要—
B1通知先メールアドレス（値）管理者のメールアドレス。複数の場合はカンマ区切り。soumu@city.example.lg.jp,hisho@city.example.lg.jp
A2システム名（ラベル）変更不要—
B2システム名（値）メール件名に使用するシステム名。会議室予約申請システム

④ GAS のウェブアプリとしてデプロイ

  1. 1
    スクリプトエディタ右上の 「デプロイ」→「新しいデプロイ」 をクリック
  2. 2
    種類の選択で 「ウェブアプリ」 を選択
  3. 3
    以下の設定を行う：
    ・説明：任意（例：会議室予約申請システム v1.0）
    ・次のユーザーとして実行：自分
    ・アクセスできるユーザー：全員（組織内のみに限定する場合は「自分と同じドメイン内の全員」を選択）
  4. 4
    「デプロイ」をクリックし、表示された ウェブアプリの URL をコピー
⚠️
コードを変更したら再デプロイが必要：GAS コードを修正した場合は「デプロイ」→「デプロイを管理」→「編集（鉛筆アイコン）」→バージョンを「新バージョン」にして「デプロイ」し直してください。同じURLで更新されます。

⑤ onEdit トリガーの設定

「申請データ」シートを管理者が編集（ステータス変更等）したときに「公開データ」シートを自動更新するためのトリガーを設定します。

  1. 1
    スクリプトエディタの左メニューから 「トリガー（時計アイコン）」 を開く
  2. 2
    右下の 「トリガーを追加」 をクリック
  3. 3
    以下の設定を行う：
    ・実行する関数：onEditTrigger
    ・実行するデプロイ：Head
    ・イベントのソース：スプレッドシートから
    ・イベントの種類：編集時
  4. 4
    「保存」をクリック。権限確認が表示された場合は許可する。
ℹ️
なぜ onEdit ではなく onEditTrigger なのか：GAS の単純な onEdit 関数はシンプルトリガーとして動作しますが、スプレッドシートへの書き込み権限を持ちません。インストーラブルトリガーとして onEditTrigger 関数を登録することで、適切な権限で動作します。

⑥ HTML ファイルへの GAS URL 設定

meeting-room-form.html をテキストエディタで開き、冒頭の GAS_URL 定数にデプロイURLを貼り付けます。

JavaScript（meeting-room-form.html 内）
// ★ ここにデプロイURLを貼り付ける
const GAS_URL = "https://script.google.com/macros/s/AKfy.../exec";
✅
GAS_URL が空のままでも動作確認できます：未設定の場合はモックデータが表示され、送信もコンソールログのみで完了扱いになります。開発・デザイン確認時に活用してください。

5複数日選択機能

日時入力エリアの下部にある「＋ 複数日を選択する場合はこちら」リンクをクリックすると複数日モードが展開されます。

複数日モードの操作手順

  1. 1
    「＋ 複数日を選択する場合はこちら」 をクリック → カレンダーUIが展開し、単日入力が非表示になる
  2. 2
    共通の時間（開始・終了時刻）を設定する（デフォルト：09:00〜17:00）
  3. 3
    カレンダー上で日付をクリックして選択（青塗りになる）。もう一度クリックで解除。過去日はクリック不可。
  4. 4
    カレンダー下の選択日リストで、日ごとに時刻を個別上書き可能。「✕」ボタンで個別削除。
  5. 5
    共通時間を変更すると、選択済み全日の時刻が一括上書きされる（個別設定も上書きされるので注意）
  6. 6
    「← 1日のみに戻す」で通常モードに戻る（選択した日付はすべてリセットされる）

  "dept":      "総務課",

  "role":      "課長",

11管理者の日常操作

ステータスを変更する

  1. 1
    「申請データ」シートを開き、該当行のK列（ステータス）をクリック
  2. 2
    プルダウンから「予約確定」または「申請却下」を選択
  3. 3
    自動的に onEditTrigger が発火し、「公開データ」シートが更新される
  4. 4
    フロントで「更新」ボタンを押すと最新の予約状況が表示される

備考を記入する

「申請データ」シートのL列（備考）に自由にメモを記入できます。フロントには表示されません。備考列の編集も onEditTrigger が発火しますが、公開データには備考列は含まれないため実害はありません。

13トラブルシューティング

予約状況が表示されない・エラーになる

症状原因対処
「データの取得に失敗しました」と表示GAS_URL が未設定または誤りHTMLのGAS_URL定数を確認。デプロイURLをコピペし直す。
コンソールにCORSエラーGASのデプロイ設定でアクセスが制限されている「アクセスできるユーザー」を「全員」に変更して再デプロイ。
送信後も「確認中」のまま更新されないonEditTrigger が設定されていないトリガー設定を確認し、onEditTrigger 関数を編集時トリガーとして登録する。
公開データシートが空のままsyncPublicData が未実行申請データのステータスを一度変更して保存し、手動でトリガーを発火させる。

送信してもスプレッドシートに記録されない

症状原因対処
シートに行が追加されないシート名が一致しないスプレッドシートのシート名（タブ名）が「申請データ」と完全一致しているか確認。
GASのログにエラーが出るinitSheets を実行していないスクリプトエディタで initSheets を手動実行する。
権限エラーが出るGASの権限が付与されていないinitSheets 実行時の権限承認をやり直す。

メール通知が届かない

原因対処
設定シートB1のメールアドレスが空B1セルに通知先メールアドレスを記入する。
メールアドレスの書式が誤りカンマ区切りは半角カンマ。スペースが混入していないか確認。
GmailApp の1日の送信上限（100通）に達しているGoogle Workspace アカウントへの切り替えを検討（1500通/日）。
スパムフォルダに振り分けられているメールクライアントのスパムフィルタを確認する。

14カスタマイズガイド

会議室の名称・数を変更する

HTMLの以下の箇所を編集します。

HTML（meeting-room-form.html）
<!-- 会議室チェックボックス -->
<label class="checkbox-item">
  <input type="checkbox" name="room" value="委員会室">  <!-- ← value を変更 -->
  <span class="check-box"></span>
  <span class="check-label">委員会室</span>           <!-- ← 表示名を変更 -->
</label>

アクセントカラーを変更する

CSS変数を変更するだけで全体の色調が変わります。

CSS（:root セクション）
:root {
  --primary: #1a4f7a;       /* メインカラー（青系） */
  --primary-light: #2563a8; /* ホバー時など */
  --accent: #2a9d8f;        /* アクセントカラー（緑系） */
}

通知メールの件名・本文を変更する

GAS の sendNotificationEmail() / sendNotificationEmailMulti() 関数内の subject 変数・body テンプレートリテラルを直接編集してください。変更後は再デプロイが必要です。

フロントをウェブサーバーではなくローカルで使う場合

ℹ️
本システムは単一HTMLファイルのため、ファイルをダブルクリックしてブラウザで開くだけで動作します。ウェブサーバーは不要です。庁内ファイルサーバーや SharePoint・Google Drive 上に置いてリンクで共有する使い方も可能です。
