# 弁当注文システム

職場内の弁当注文を受け付け、Google Apps Script と Google スプレッドシートで注文を管理するシステムです。

## ファイル構成

- `index.html`: 注文フォームと現在の注文状況表示
- `Code.gs`: 注文POST受付、公開注文状況GET、マスタJSON出力
- `Hanei.gs`: 管理者用 `orders` シートから公開用スプレッドシートへの同期
- `data/masters.json`: フォーム表示用のメニュー・サイズマスタ

## 注文保存

`index.html` は `GAS_URL` にPOSTし、`Code.gs` の `doPost(e)` が管理者用スプレッドシートの `orders` シートへ保存します。

`orders` シートの列は以下です。

```text
タイムスタンプ / 部署 / 名前 / 連絡先 / メニュー / サイズ / 基本単価 / サイズ加算額 / 単価 / 数量 / 状態 / 備考
```

注文保存後、可能であれば `syncPublicSheet()` を呼び出して公開用シートも更新します。公開同期に失敗しても、注文保存が成功していれば送信成功として扱います。

## 公開注文状況

公開用スプレッドシートには、状態が `有効` の行だけを同期します。

公開対象列は以下だけです。

```text
タイムスタンプ / 部署 / メニュー / サイズ / 基本単価 / サイズ加算額 / 単価 / 数量
```

以下の情報は公開用スプレッドシート、GETのJSON、HTMLの注文状況表示に含めません。

```text
名前 / 連絡先 / 状態 / 備考
```

`Code.gs` の `doGet(e)` は公開用シートを読み取り、注文一覧と更新日時をJSONで返します。`subtotal` は `unitPrice * quantity` で計算します。

## 画面仕様

注文フォーム下部に「現在の注文状況」を表示します。通常時は折りたたまれており、「注文状況を表示」を押すと展開できます。

展開後は以下を確認できます。

- 全体サマリー: 合計数量、参考合計金額、注文行数
- 部署フィルター
- メニュー別集計
- 部署別集計
- 注文一覧

注文送信に成功すると、注文状況を自動で再取得します。

## 設定

`Hanei.gs` の `PUBLIC_SPREADSHEET_ID` に公開用スプレッドシートIDを設定してください。URL全体を入れてもID部分を抽出して利用します。

`index.html` の `GAS_URL` には、デプロイ済みGAS WebアプリのURLを設定してください。

## 手動テスト

1. GASに `Code.gs` と `Hanei.gs` を反映する。
2. `PUBLIC_SPREADSHEET_ID` が公開用スプレッドシートを指していることを確認する。
3. GASエディタで `syncPublicSheet()` を実行し、公開用シートの列が公開対象8列だけになることを確認する。
4. `doGet(e)` のWebアプリURLへGETし、JSONに `名前`、`連絡先`、`状態`、`備考` が含まれないことを確認する。
5. `index.html` を開き、「現在の注文状況」が初期状態で折りたたまれていることを確認する。
6. 「注文状況を表示」を押し、集計と一覧が表示されることを確認する。
7. 注文を送信し、送信成功後に注文状況が再取得されることを確認する。
8. `orders` シートの状態を `取消` や `非公開` に変えて `syncPublicSheet()` を実行し、公開用シートから除外されることを確認する。
