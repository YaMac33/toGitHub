更新手順書

このサイトは、共有フォルダ内のHTMLファイルをそのまま開いて見る 「オフライン・完全ローカル」のサイトです。インターネットにはつながっていません。 このフォルダを丸ごと共有フォルダに置き、index.html を開けば表示されます （ブラウザは Microsoft Edge を想定しています）。

更新担当者がさわるのは、基本的に 記事HTML と layout.js の記事リスト の2か所だけです。 それ以外は原則さわりません。

新しい記事を追加する手順

全体で3ステップです。順番どおりに進めてください。

ステップ1　雛形をコピーして記事を作る

1. _template フォルダの「記事テンプレート.html」をコピーする
2. コピーしたファイルを pages フォルダに移動する
3. ファイル名を、次のルールで付け直す

ファイル名のルール： 日付_内容.html（すべて半角）
例：2026-08-01_meeting-room.html
・日付は YYYY-MM-DD の形（2026-08-01 など）
・内容は半角の英数字とハイフンだけ（日本語やスペースは使わない）

ステップ2　記事の中身を書く

4. pages に置いたファイルを右クリック →「プログラムから開く」→「メモ帳」で開く
5. ファイルの中の 【書き換え1〜4】 と書かれた場所だけを書き換える （それ以外はさわらない）
6. 上書き保存する

ステップ3　一覧に載せる（いちばん大事）

7. layout.js をメモ帳で開く
8. 上のほうにある「記事リスト（ARTICLES）」を探す
9. 次のかたまりを1つコピーして、すぐ上に貼り付ける 

{
  file: "2026-07-02_dx-results.html",
  title: "DX推進リーダー これまでの取り組みと実績",
  date: "2026-07-02",
  category: "",
  body: "記事の本文をここに貼り付ける"
},

10. 貼り付けたほうの中身を、今回の記事に書き換える

fileステップ1で付けたファイル名（正確に）
title一覧に表示したいタイトル
date日付（YYYY-MM-DD）
category空っぽ（"" のまま）でOK
body記事本文のテキスト。検索に使われます（下の注意を参照）

11. 上書き保存する
12. index.html を開いて、一覧に新しい記事が出ていれば完成です

カンマ「,」に注意： 各かたまりの最後（閉じカッコ } の後ろ）にカンマが必要です。 付け忘れると一覧が表示されなくなります。 いちばん最後の記事だけはカンマ不要です（元からあるお手本を参考に）。

検索機能と「body」について

トップページには記事を横断検索する機能があります。検索は、記事リストの body に書かれたテキストを対象にしています。

大事なルール： 記事の本文（pages のHTML）を後から直したときは、 layout.js の body も同じ内容に直してください。
片方だけ直すと、検索結果と実際の記事の中身がズレてしまいます。 （システムの都合で、本文と body は自動では同期されません）

※ 検索は、英数字の全角・半角、大文字・小文字は自動で区別せずにヒットします。 ただし、ひらがなとカタカナ（例：「システム」と「しすてむ」）は別の文字として扱われます。

画像を入れたいとき

1. 画像ファイルを images フォルダに入れる
2. ファイル名は次のルールで付ける（すべて半角）

日付_記事名_連番.jpg
例：2026-08-01_meeting-room_01.jpg

3. 記事HTMLの中の【任意：画像を入れる場合】の説明にしたがって、 画像の行を有効にし、ファイル名を書き換える

うまく表示されないとき

症状確認すること
一覧に記事が出ない layout.js のカンマ「,」の付け忘れ・付けすぎがないか確認
検索しても出てこない その記事の body にテキストが入っているか確認。 ひらがな／カタカナの違いでもヒットしません
見た目が崩れる（文字だけになる） style.css / layout.js が index.html と同じ場所にあるか。 記事は pages、固定ページは info に入っているか （場所が変わると参照がずれて読み込めません）
画像が出ない images フォルダに画像があるか、ファイル名が一致しているか （大文字小文字・全角半角も区別されます）

さわってはいけないファイル

  • style.css（見た目全体の設定）
  • layout.js の「記事リスト」以外の部分

これらは全ページ共通です。壊すと全ページに影響します。 変更が必要なときは、DX推進担当（〇〇）に相談してください。
