PROMPT

記事生成プロンプト早見表

AIに記事を作らせるときに使う早見表とプロンプトの型です。 各ブロックの「コピー」ボタンで全文をコピーできます。

使い方：まず ①デザインシステム早見表 を貼り、続けて ②プロンプトの型 を貼って、下部の「この記事の情報」を埋めて送信します。

① デザインシステム早見表

毎回、最初に貼るもの。

[デザインシステム参照ルール]
・style.css / layout.js は変更禁止。既存のクラス・トークンのみで組む。
・インラインstyleでの色/フォント直指定は禁止。必ず変数(var(--color-xx)等)を使う。
・個人名・団体名・固有名詞は 〇〇 で伏せる(実データは後で自分で埋める)。
・ページ固有JSは<body>末尾にIIFEで記述。

■ 共通レイアウト(layout.js)
<site-header base="."></site-header>   … ルート(index.html)の場合
<site-header base=".."></site-header>  … pages/ や info/ 内の場合
<site-footer></site-footer>            … owner属性省略可(既定値あり)
<article-list base="."></article-list> … 記事一覧を自動生成(トップページ用)
※ base はページの階層に合わせて "." か ".." を指定する
※ CSS/JSの読み込みパスも同じ考え方(ルート="./style.css" / pages・info内="../style.css")

■ 色トークン
--color-ink / --color-ink-soft   : 主要/補助テキスト
--color-paper / --color-paper-raised : 背景/カード面
--color-mist                     : 罫線・区切り
--color-brass / --color-brass-soft : アクセント(強調・シグネチャ)
--color-slate / --color-slate-soft : セカンダリ(リンク等)
--color-success/warning/danger (+soft) : 状態色

■ レイアウト
.container / .flex / .flex-col / .grid .grid-2 .grid-3
.stack (子要素に自動余白) / .divider
gap-1〜6, .items-center, .justify-between

■ タイポグラフィ
h1〜h4(明朝) / p(ゴシック) / .text-sm .text-xs .text-lg .text-muted
.section-title + <span class="eyebrow"> (シグネチャ見出し)
.article-body (記事本文の幅・行間調整) / .article-meta (記事の日付表示)

■ コンポーネント
.btn-primary / .btn-accent / .btn-ghost
.card / .card-accent
.form-group / .label / .input / .select / .textarea / .hint
.table
.badge-success/warning/danger/neutral
.alert-success/warning/danger

② 毎回使うプロンプトの型

①のあとに貼り、「この記事の情報」を埋めて送信する。

[役割]
デザインシステム(style.css / layout.js)に準拠して、新しい記事1本を作成してください。

[前提]
・出力は3点セット
   ① pages/ 用の記事HTML本体(<main class="container article-body">以下)
   ② layout.js の ARTICLES に追記する1ブロック(file/title/date/category/body)
   ③ ファイル名(YYYY-MM-DD_内容.html、半角英数字)の提案
・pages/内のHTMLなので、CSS/JS参照と<site-header>のbaseは "../" ".." を使う
・個人名・団体名・固有名詞は 〇〇 で伏せる(実データは後で自分で埋める)
・②の body には、記事本文のテキストを検索用に入れる(本文と同じ内容にする)

[この記事の情報]
・日付:
・タイトル:
・目的/種別: (実績報告 / お知らせ / 活動レポート など)
・本文で伝えたいこと(箇条書きでOK):
・表や画像は使うか: (使う場合は概要も)
※ 検索機能を追加したため、②の出力には body（検索用の本文テキスト）が 含まれます。記事本文を直したときは body も直してください。
