# 初期設定チェックリスト

## Googleカレンダー

- [ ] 予約管理用のGoogleカレンダーを作成した
- [ ] カレンダーIDをコピーした
- [ ] 管理者が予定を作成・削除できる権限を持っている
- [ ] カレンダーのタイムゾーンが日本時間になっている

## Googleフォーム

- [ ] `form-fields.md` の項目を作成した
- [ ] 必須項目を設定した
- [ ] 回答先スプレッドシートを作成した
- [ ] 回答先シート名を `申請一覧` にした
- [ ] フォーム送信後の確認メッセージを設定した

## スプレッドシート

- [ ] フォーム回答列が想定通りに作成されている
- [ ] 管理列を追加した
- [ ] `ステータス` 列に `未処理`、`承認`、`却下`、`取消` のプルダウンを設定した
- [ ] 管理者以外が管理列を編集できないように保護した
- [ ] テスト申請を1件送信した

## Apps Script

- [ ] `Code.gs` を貼り付けた
- [ ] `Index.html` を作成して貼り付けた
- [ ] `Style.html` を作成して貼り付けた
- [ ] `Script.html` を作成して貼り付けた
- [ ] Apps Scriptのタイムゾーンを `Asia/Tokyo` にした
- [ ] `CONFIG.CALENDAR_ID` を変更した
- [ ] 必要に応じて `CONFIG.CANCEL_MODE` を変更した
- [ ] 必要に応じて `VIEW_CONFIG` を変更した
- [ ] `setupSheet` を手動実行して権限承認した

## トリガー

- [ ] `handleFormSubmit` を「フォーム送信時」で設定した
- [ ] `handleStatusEdit` を「編集時」で設定した
- [ ] トリガー実行ユーザーがカレンダー編集権限を持っている
- [ ] トリガー実行ユーザーがメール送信可能である

## Webアプリ

- [ ] Webアプリとしてデプロイした
- [ ] 実行ユーザーを「自分」にした
- [ ] アクセス範囲を運用方針に合わせて設定した
- [ ] WebアプリURLを開いて予約状況が表示されることを確認した
- [ ] 月、週、日、一覧の切替を確認した
- [ ] 利用場所フィルターを確認した
- [ ] 日付範囲フィルターを確認した

## Googleサイト

- [ ] トップページを作成した
- [ ] 予約状況ページにGAS WebアプリURLを埋め込んだ
- [ ] 予約申請ページにGoogleフォームを埋め込んだ
- [ ] 利用ルールページを作成した
- [ ] 管理者向けページを作成した
- [ ] 公開範囲を確認した

## 動作確認

- [ ] フォーム送信後、ステータスが `未処理` になる
- [ ] フォーム送信後、受付メールが届く
- [ ] `承認` に変更するとGoogleカレンダーに予定が作成される
- [ ] `承認` に変更すると承認メールが届く
- [ ] 同じ場所・同じ時間帯の重複予約がエラーになる
- [ ] `却下` に変更すると却下メールが届く
- [ ] `取消` に変更するとカレンダー予定が削除または取消表示になる
- [ ] 同じステータスを再編集しても二重メールや二重イベントが発生しない
