システム概要

本システムは、ブラウザ上で動作するWebアプリケーション（所属内の権限がある方のみ閲覧可能）です。画面の表示には、以下の3つの要素を使用しています。

  • HTML: 画面の構造や文章を定義するベースとなるファイルです。
  • CSS: 文字の大きさや色など、画面の見た目を整えます。
  • JavaScript (JS): 画面に動きや便利な機能を追加します。

※CSSとJSはHTML内に直接書くこともできますが、本システムではそれぞれ別のファイルとして保存し、HTMLから読み込んで連携させています。

ファイルの取り扱いに関する注意点

  • ファイルの直接編集について
    Webページのような見た目ですが、実態はテキストファイルです。「右クリック → プログラムから開く → メモ帳」などで開けば、コードの閲覧や編集も可能です。ただし、誤って書き換えるとシステムが壊れるリスクがあるためご注意ください。
  • 移動時の注意（フォルダごと移動）
    本システムは、複数のファイルが特定のフォルダ階層（位置関係）を保つことで動作しています。システムを別の場所に移動させる際は、ファイル単体ではなく「フォルダごと」移動してください。

データの更新手順

システムの元データは、既存のExcelファイルです。入力項目が無駄に増えたり複雑になったりしないよう工夫しつつ、以下の加工を加えています。

  • アプリ更新用の列の追加
  • Excelのデータを抽出し、システム表示用のデータ（jsファイル）を自動作成するVBA（マクロ）の追加
＜更新方法＞

Excelファイルを更新した後、「開発」タブ ＞ 「マクロ」 ＞ 「（該当のマクロ名）」を選択して実行してください。
これにより、表示用のデータ（jsファイル）が丸ごと最新の状態に上書きされます。

本システムを作成した目的

従来のExcel管理では難しかった「複数の項目の関連性を体系的に見る」ことを実現するためです。

また、「編集するファイル（Excel）」と「閲覧するファイル（ブラウザ）」を完全に分離したことで、検索性が向上するだけでなく、閲覧者が誤ってデータを壊してしまうリスクを低減し、安全かつ見やすいデータベースとして利用できるようになっています。
