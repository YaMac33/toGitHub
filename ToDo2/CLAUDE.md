# CLAUDE.md

このリポジトリで Claude Code が作業する際の規約とプロジェクト仕様。

## プロジェクト概要

複数の CSV を読み込み、日時順に統合表示する **ローカル完結型 TODO リスト** を作る。
サーバー不要・ビルド不要で、`index.html` をブラウザで直接開く（`file://`）だけで動作することが最重要要件。

## 絶対要件（崩してはいけない制約）

- **単一ファイル構成**: HTML / CSS / JavaScript をすべて `index.html` 1ファイルに収める。
- **ビルドツール・外部ライブラリ禁止**: npm / バンドラ / CDN 依存を一切使わない。素の HTML+CSS+Vanilla JS のみ。
- **`file://` で動作**: `fetch()` は使わず、ファイル読み込みは `<input type="file">` と `FileReader` で行う（ローカルで開いた際の CORS 制約を回避するため）。
- **オフライン動作**: 外部ネットワークアクセスを行わない。
- **LLM 保守性**: 非エンジニアの後任者が将来 LLM に渡して改修できるよう、コードは平易・コメント付き・関数を小さく保つ。

## 機能仕様

1. **複数 CSV 読み込み**: ファイル選択（`multiple`）とドラッグ＆ドロップの両方に対応。
2. **ファイル単位の表示/非表示**: 読み込んだファイルごとにチェックボックスを置き、オフにするとそのファイル由来のタスクのみ一覧から隠す（データはメモリ上に保持しフィルタするだけ）。
3. **統合・ソート表示**: 表示中ファイルのタスクを結合し、日時で昇順/降順を切り替えてソート表示。
4. **完了/未完了の分離**: 未完了を上、完了を下部の別セクションに表示。完了行は取り消し線。チェック操作で即座にセクション間を移動。
5. **保存（上書き）**: 変更があったファイルにのみ「未保存」表示を出し、保存ボタンで**変更のあったファイルだけ**を**元のファイル名のまま**個別ダウンロード（`Blob` + `<a download>`）。

## CSV フォーマット

ヘッダー行 + データ行。列順は固定。

```
日時,タスク内容,完了
2026-06-15 10:00,会議資料の準備,未完了
2026-06-16 14:30,議会答弁書チェック,完了
```

- 日時: `YYYY-MM-DD HH:MM` を基本とし、`YYYY/MM/DD HH:MM`（スラッシュ区切り）も解釈できること。
- 完了列: 文字列 `完了` / `未完了`。`完了` のみを done=true とみなす。
- 文字コードは UTF-8。書き出し時は Excel での文字化けを避けるため **BOM 付き UTF-8** とする。

## 実装上の注意

- **CSV パースは自前実装**: ダブルクォートで囲まれたフィールド（カンマ・改行・エスケープ `""` を含む）を正しく処理すること。安易な `split(",")` は不可。
- **不正・空の日時**: パースできない日時はソート時に末尾へ送る（`Infinity` 等）。クラッシュさせない。
- **CRLF / LF 混在**: 読み込み時に改行コードを正規化する。
- **行の出自を保持**: 各タスクはどのファイルの何行目由来かを保持し、保存時に元ファイルを正しく再構築できること。
- **状態はメモリのみ**: localStorage には保存しない（保存はユーザーの明示的なダウンロード操作で行う方針）。

## デザイン方針

- 自治体・行政事務での実務利用を想定。落ち着いた信頼感のある配色（ペーパーホワイト基調＋藍系アクセント）。
- 機能を邪魔しない実用本位。過剰な装飾・アニメーションは避ける。
- レスポンシブ対応（スマホ幅で「ファイル」列を畳むなど）。キーボードフォーカスが見えること。
