# Claude Code に最初に投げる指示プロンプト

以下をそのまま Claude Code のチャットに貼り付けてください。
（`CLAUDE.md` をリポジトリ直下に置いた状態で実行することを前提にしています）

---

このリポジトリの `CLAUDE.md` に記載の仕様に従って、ローカル完結型の TODO リスト Web アプリを新規に実装してください。

進め方:

1. まず `CLAUDE.md` を読み、要件・制約・受け入れ確認の項目を把握する。
2. 実装計画（ファイル構成、データ構造、主要関数の責務）を箇条書きで提示する。
3. 合意が取れたら `index.html` を実装する。CSV パースは引用符フィールドに対応した自前実装にすること。
4. 動作確認用に `samples/todo_sample1.csv` と `samples/todo_sample2.csv`（日時が前後し、完了/未完了が混在し、カンマ入りの引用符フィールドを1つ含む）を作成する。
5. パースとソートのロジックは、Node で動く簡易テストスクリプトを書いて検証し、結果を示してから本体に反映する（テストスクリプトは最終成果物には含めない）。
6. `README.md` に、CSV フォーマット・ブラウザでの開き方・保存の挙動（変更ファイルのみ元の名前で個別ダウンロードされる点）を日本語で記載する。
7. 最後に `CLAUDE.md` の「受け入れ確認」リストを1項目ずつ自己点検し、結果を報告する。

制約の再確認（特に重要）:

- 単一ファイル（`index.html`）に HTML/CSS/JS をすべて収める。
- 外部ライブラリ・CDN・ビルドツールは使わない。
- `fetch()` は使わず `FileReader` でファイルを読む（`file://` で動かすため）。
- 完了状態は localStorage に保存せず、保存はダウンロードで行う。
- 書き出す CSV は BOM 付き UTF-8。

不明点があれば実装前に質問してください。勝手に仕様を変更せず、変更が必要だと判断した場合は理由を添えて提案してください。
