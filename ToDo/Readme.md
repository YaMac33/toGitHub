# 業務ToDo（着手可能 / 待ち 管理）

締切順ではなく「着手できるようになった順」でタスクを表示する単一ファイルWebアプリ。
着手可能の条件は「条件なし／前提タスク完了後／指定日以降」の3種を組み合わせて設定でき、
条件を満たしたタスクは自動的に「待ち」から「着手可能」へ上がる。

## 構成

| ファイル | 役割 | 置き場所 |
|---|---|---|
| `index.html` | アプリ本体（単一ファイル） | GitHub Pages |
| `data.js` | タスクデータ（プレーンなJS） | 共有フォルダ |

## セットアップ

1. このフォルダを GitHub リポジトリにして push する。
2. リポジトリの Settings → Pages で公開する（`index.html` が配信される）。
3. `data.js` を庁内の共有フォルダに置く。
4. Chrome または Edge で Pages のURLを開き、「データを開く」で共有フォルダの `data.js` を選んで書き込みを許可する。
5. 以降は「保存」で `data.js` がその場で上書きされる。次回は「前回のファイルに再接続」でワンクリック復帰。

初回は「サンプルで試す」で動作を確認 →「ダウンロード」で `data.js` を作成して共有フォルダに置く、という始め方も可。

## ローカルで動かす（開発時）

File System Access API は https / localhost でのみ有効。`file://` では保存できないため、簡易サーバーで配信する。

```
npx serve .
```

## 注意点

- 対応ブラウザは **Chrome / Edge** のみ（Firefox・Safari は表示とダウンロードのみ）。
- ネットワークドライブだと書き込みが弾かれる場合がある。実フォルダで保存可否を確認する。
- 同時編集は後勝ち（最後の保存で上書き）。
- `data.js` をリポジトリに含めて公開すると中身が誰でも閲覧できる。機微な内容は共有フォルダ運用にする。
